## 关键发现

### 跨批共同模式
1. **Premise质疑系统性缺失** → 事后补救（所有批次）
2. **多模型价值分化**：代码Bug靠独立性，推理错误靠多样性（Batch 1+2）
3. **失败→方法论资产的转化流水线**：诊断→命名→模板化→可迁移（所有批次）
4. **证据等级意识自然收敛**：A/B/C/D分级在不同项目自发一致

### 提取格式演进
```
嵌入式(LIT/AgentHarness) → 独立文档(SmallScale) → 四维度(GitNexus) → 
五组件+缺口矩阵(PilotDeck) → 证据校准+六步闭环(Evolver)
```

### 提取质量排名（Batch 2）
1. Evolver (9.4/10) — 证据可靠性天花板
2. PilotDeck (8.8/10) — 格式完备性天花板
3. Small Scale (7.8/10) — 最干净的独立提取
4. GitNexus (7.8/10) — 反模式维度创新
5. LIT (5.0/10) — 信息密度最高但无结构
6. Agent Harness (3.0/10) — 提取成熟度最低

### Batch 1 vs Batch 2 本质差异
- 质量控制：统计推断 vs 多模型审查收敛
- 产出性质：创造知识 vs 评估知识
- 不确定性来源：市场噪声 vs AI判断噪声
- 提取成熟度：Batch 1显著低于Batch 2（无独立提取文档）

### 关键风险
- **零人类独立审查**：120+片段全部由AI提取和评估
- **自利偏差**：CLAUDE.md和评估分析均AI生成
- **证据深度不均**：大项目(ETF/Evolver/LIT)远深于轻量项目
- **BDC2026活跃污染**：已按边界案例处理
