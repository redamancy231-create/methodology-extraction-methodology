## 一、总判断

Prompt-TDD Methodology 可提取出一套结构明确、可迁移性较高的 Prompt 对照实验方法论。其核心不是“怎样写出更长、更复杂的 prompt”，而是把 prompt 改动转化为可证伪实验：先冻结假设、因变量、测试集和停止规则，再以 Tier 0 工程门验证管线，以 Tier 1 科学门判断效应，最后通过异后端独立审查和成熟度标注闭合；即使结果为阴性，也保留数据、失败原因和方法论资产。

本项目最有价值的提取物可概括为：

> **候选模式发现 → 可证伪化 → 预注册冻结 → 小样本管线验证 → 推断检验 → 异后端盲审 → 发现修复与复验 → 阴性结果闭合 → 限域写回框架。**

证据等级评为 **B**，而非 A。原因是目标仓库、上游实验仓库、Git 历史及运行结果提供了多项一手证据，但发布包存在显著的版本/溯源漂移：A1 状态陈述已经过时，A2/A3 随附数据与通用分析脚本不兼容，预注册锁和部分原始数据未随案例手册发布，“17+ 轮”的计数口径也随文档版本变化。

设计动机可由跨项目证据链还原：

3. 因此，项目不是凭空提出三种 prompt 形式，而是在看到外部代码架构后，把“代码层面的优秀设计是否能迁移成 prompt 层面的可检测优势”转化为对照实验。

由此可提取一条重要原则：

> **不要因为某种代码架构在工程上优雅，就直接把它升级为 prompt 最佳实践；先将其操作化为独立变量，并用受控实验验证。**

三项实验的一致阴性观察进一步形成限定性结论：在 GPT-5.5、temperature=0 和相应中文结构化任务域中，代码架构优势没有自动转化为可检测的 prompt 性能优势；这不等于否定这些架构在代码可维护性、模块化或复用性上的价值。

## 六、最终提取的方法论组件

### M1 测试先于 prompt

先定义“什么算正确”和失败用例，再寻找能通过测试的 prompt。新失败进入测试集，而不是只向 prompt 追加补丁规则。

### M2 发现—验证—集成三层流水线

外部项目分析负责产生候选模式，Prompt-TDD 负责验证，框架只接收带证据等级和适用域的结论。发现本身不能直接升级为规范。

### M3 CK1-CK6 事前门控

- CK1：假设可证伪；
- CK2：因变量可操作化；
- CK3：功效和最小兴趣效应量；
- CK4：双臂 INVENTORY 等价，防止 IV 污染；
- CK5：train/test 分离；
- CK6：天花板探测。

### M4 预注册锁

在执行前冻结假设、分析方法、效应量阈值、停止规则和 test set hash。任何改变必须留下新的 commit 轨迹。

### M5 工程门与科学门分离

- 工程门回答“管线能不能用”；
- 科学门回答“假设是否得到支持”。

`工程门 PASS + 科学门 FAIL` 是有效阴性证据，不是项目失败。

### M6 Tier 0 先止损，Tier 1 再推断

Tier 0 用最小样本验证采集、评分和检测空间；只有管线通过、没有严重天花板/DV 退化时才投入 Tier 1。A3 v2 的“不执行”是该原则的典型产物。

### M7 异后端盲评与审查

评分者、实验执行者和独立审查者尽量跨后端分离；解盲前不暴露组别和作者自评。审查被视作测量仪器校准，而不是修辞性背书。

### M8 阴性结果成熟度

使用 `[Sp] → [E] / [E-] → [F·1域] → [J]` 标注证据成熟度。天花板、低功效或设计不能检验假设时，用 `[E-]` 和限制语句闭合，禁止包装成“证明无效”。

### M9 发现—修复—复验三态闭环

每项发现必须进入：修复并复验、保留并披露、阻断并停止。修复后结论是否变化也必须记录。

### M10 数据与结论双向溯源

理想链路为：候选资产 → 设计文档 → prompt 双臂 → test set/hash → raw outputs/manifest → scoresheet → 分析报告 → 审查记录 → 修复 commit → 成熟度与框架写回。目标仓库已经实现此链路的大部分概念，但尚未在发布包中完整落地。

## 八、结论

Prompt-TDD Methodology 的方法论核心成立，且优于普通“经验型 prompt engineering”的地方在于：它把主观改写转化为可证伪、可停止、可审查、可诚实失败的实验流程。最有价值的产出并不是某个结构化 prompt 被证明更好——事实上 A1/A2/A3 都未产生稳定阳性信号——而是建立了如何发现无效实验、如何避免 HARKing、如何区分工程成功与科学失败、以及如何公开阴性结果的操作协议。

但作为发布级案例手册，它当前还不能评为“完整提取/A 级证据”：A1 状态漂移和真实案例复现失败是两个必须修复的硬缺口。综合判定为：**7/8 组件可主要提取，证据等级 B，提取层级“主要提取”。**
