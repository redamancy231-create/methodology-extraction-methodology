## 修正2: MUST 三层化

Codex 指出 10/11 项 MUST 过重，会导致未来不遵守。改为三层：

### Layer 1: 核心字段（每份正式提取必有，6项）

| # | 字段 | 说明 |
|---|------|------|
| F1 | **元数据 / Provenance** | 生成模型(非CLI壳名)、日期、源项目路径、提取者上下文 |
| F2 | **优先级 / 采纳状态** | 每片段标注 P0/P1/P2/P3 或 Adopt/Modify/Reject |
| F3 | **证据等级 + 定义** | 每片段标注 A/B/C/D/[Sp]；文档首部定义等级含义 |
| F4 | **迁移条件 / 反例** | P0/P1 片段强制完整迁移条件（适用场景+前提+边界+失败模式+成本）；P2/P3 可用轻量字段 |
| F5 | **局限与源可信度** | 提取文档整体局限声明 + 源项目可信度评估（不混入提取质量分） |
| F6 | **反模式 / 警示** | 独立章节或至少标注"未发现反模式" |

### Layer 2: 发布 Gate（发布前必须通过，2项）

| # | Gate | 说明 |
|---|------|------|
| G1 | **独立审查** | ≥1轮异后端审查，阻断项已处理 |
| G2 | **md+json 双件** | 同步交付，内容一致性已验证 |

### Layer 3: 可选附录（按需，4项）

| # | 附录 | 触发条件 |
|---|------|---------|
| A1 | 修订历史 | 多版本迭代时 |
| A2 | 提取者偏误声明 | 提取模型已知有系统性倾向时 |
| A3 | 排除清单 | 源项目中有观察到但决定不提取的元素时 |
| A4 | 跨项目模式识别 | 累积≥3个项目提取后 |
